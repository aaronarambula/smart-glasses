# Repository Structure

This document describes the organization of the Smart Glasses codebase.

## Root Level

```
smart-glasses/
├── README.md                  # Project overview and quick start
├── CMakeLists.txt            # Build configuration
├── aaronnet_risk.bin         # Trained neural network weights
├── .gitignore                # Git ignore rules
├── .github/                  # GitHub-specific files
├── docs/                     # Documentation
├── scripts/                  # Testing and utility scripts
├── build/                    # Build output (generated)
└── [Source directories]      # See below
```

## Building

```bash
mkdir -p build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release -DUSE_SIM=ON
cmake --build . --parallel
```

All targets build into `build/app/` and `build/[library_name]/`.

## Running

```bash
# Main application
./build/app/smart_glasses [--sensor sim|ld06|rplidar] [options]

# Utilities
./build/app/export_risk_model --checkpoint aaronnet_risk.bin --inspect
./build/app/extract_features --num-frames 100 --export features.csv

# Tests
bash scripts/end_to_end_system.sh --quick
bash scripts/test_button_gpio.sh simulate 17 2.0
```

## Contributing Guidelines

### When Adding New Code:
1. Place in appropriate module directory (sensors/, perception/, etc.)
2. Follow existing naming conventions (header in include/, implementation in src/)
3. Update CMakeLists.txt for new source files
4. Add documentation if it's a new feature
5. Run: `bash scripts/end_to_end_system.sh --quick` to verify

### When Adding New Documentation:
1. Place in `docs/`
2. Link from main README.md if it's a primary feature
3. Update this STRUCTURE.md if needed

### When Adding New Scripts:
1. Place in `scripts/`
2. Make executable: `chmod +x scripts/my_script.sh`
3. Add to this guide

## Deployment

For Raspberry Pi deployment:
1. Build on Pi or cross-compile
2. Copy binary to `/usr/local/bin/smart_glasses`
3. Create systemd service (see README.md)
4. Run: `systemctl start smart-glasses`
